# Workflow of RQ-01:
The primary objective of RQ1 is to create a realistic federated prognostics scenario by considering sensor heterogeneity among different airline operators. Since the NASA C-MAPSS dataset is centralized and does not contain direct airline-specific information, the engines were first divided into five clusters, representing five independent airline operators. This partitioning enables the simulation of a federated learning environment.
As the dataset does not directly provide Remaining Useful Life (RUL) labels, RUL was calculated for each engine by subtracting the current cycle number from its maximum cycle before failure. After generating the RUL labels, sensor heterogeneity was introduced by assigning different subsets of sensors to different airlines, reflecting real-world situations where operators often collect different types and numbers of measurements.
However, this introduced a major challenge: different airlines now had different input dimensions, making it impossible to train a common federated model directly. To address this issue, client-specific sensor adapters were employed to transform each airline's sensor subset into a shared 64-dimensional latent representation. This latent space provides a common feature format while preserving the available information from each client.
The latent representations were then organized into sliding windows of 30 cycles, producing input sequences of shape (30, 64). Sliding windows were used to capture temporal degradation patterns, long-term dependencies, and engine health trends that cannot be observed from a single cycle. This step is particularly important because LSTM networks are designed to learn from sequential data.
These sequences were used to train a federated multi-task Long Short-Term Memory (LSTM) model. The shared LSTM backbone simultaneously performed two tasks: RUL prediction through a regression head and early fault detection through a classification head. During each federated learning round, airline operators trained the model locally using their private data and shared only model parameters with the central server. The server aggregated local updates using the Federated Averaging (FedAvg) algorithm and distributed the updated global model back to all clients.
After training, the final global model was evaluated using RMSE for RUL prediction and Accuracy, Precision, Recall, F1-score, and AUPRC for fault detection. Finally, model performance and training behavior were analyzed using federated training loss curves, actual-versus-predicted RUL plots, prediction error distributions, fault probability trends, and client-wise performance comparisons.

# WorkFlow of RQ-05:
For solving RQ-5, our main objective is to preserve the uniqueness of each airline's data while still enabling knowledge sharing among operators. In traditional FedAvg, a model trained on one airline may receive a low score when evaluated on another airline simply because their data distributions are different, not because the model itself is poor. This can unfairly reduce the contribution of certain airlines during model aggregation and lead to sub-optimal global performance.
This issue is particularly important in real-world aviation systems where airlines operate under different environmental and operational conditions. For example, airlines operating in Arctic regions may experience very different engine behavior compared to those operating in tropical climates. Similarly, short-haul and long-haul operators, as well as airlines serving different geographical regions, often exhibit distinct usage patterns, maintenance practices, and degradation characteristics. These factors naturally create non-IID data distributions across operators. Therefore, a federated learning framework should allow airlines to benefit from shared knowledge without being penalized simply because their operating conditions differ from others.
To solve this issue, each airline's data distribution is first described using latent sequence representations, and distributional disparities are measured using pairwise Maximum Mean Discrepancy (MMD). These distances are transformed into similarity scores, resulting in an airline similarity matrix. Each airline then trains its own multi-task LSTM model, similar to RQ-1, with distinct heads for RUL prediction and early failure detection. During federated learning, airlines only send model parameters to the central server. Aggregation is done using similarity-aware weights based on dataset size and inter-airline similarity, rather than the traditional FedAvg. This enables airlines to gain more from operators with comparable data distributions while retaining global knowledge. The process is repeated over multiple communication rounds, and the final airline-specific models are evaluated using RMSE for RUL prediction and Accuracy, Precision, Recall, F1-score, and AUPRC for fault detection.


Finally, prediction outcomes, fault probabilities, and assessment metrics are saved and compared to the conventional FedAvg baseline. This comparison contributes to determining if similarity-aware aggregation may improve federated prognostics performance under actual non-IID airline operational settings.
